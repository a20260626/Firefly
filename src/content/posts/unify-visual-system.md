> 功能都能跑以后，页面还需要变得像一个整体。
> 这一篇讲视觉统一：如何把首页、文章页、侧栏、正文宽度和卡片样式，调整成一套清爽的技术博客语言。

---

上一篇我们做了站内导航。

文章标题能点，继续阅读能点，Learning Stages、Tool Index、最近文章、上一篇和下一篇都能把用户带到别的文章。

这时网站已经不只是"能打开"了。

它已经有了内容站的基本能力。

但还有一个很重要的问题：

> **它看起来是不是一个统一的网站？**

如果首页是一种宽度，文章页又是另一种宽度；

如果左侧卡片很窄，右侧索引又很宽；

如果正文、标题、按钮、链接各有各的样式；

用户会感觉这是几个模块拼在一起，而不是一个完整作品。

所以这一篇要做的是视觉统一。

---

## 为什么视觉统一要放在后面

一开始就做视觉，听起来很诱人。

因为视觉最容易看到成果。

但对一个内容站来说，过早追求视觉会有两个问题：

1. 页面结构还没定，样式会反复推翻
2. 内容关系还没跑通，视觉只是漂亮空壳

所以本站的顺序是：

1. 先搭项目基础
2. 再做信息架构
3. 再做首页体验
4. 再做文章阅读页
5. 再做站内导航
6. 最后统一视觉

这不是说视觉不重要。

恰恰相反。

视觉很重要，所以它应该服务已经清楚的结构。

当首页、文章页、侧栏和导航都确定以后，再统一视觉，才不会一直返工。

---

## 视觉参考不是照抄

本站参考的是清爽技术博客风格。

参考的重点不是复制某个网站的每一个像素。

真正要学习的是它的设计判断：

- 背景保持浅灰，让正文卡片更清楚
- 主要内容用白色卡片承载
- 链接使用稳定的蓝色
- 边框和阴影都很轻
- 中间正文宽度克制
- 左右侧栏承担导航和索引
- 页面信息密度适合技术阅读

这是一种气质。

它不是炫酷，也不是营销感。

它更像一个长期写作的技术笔记站：

> 安静、清楚、可读、可持续。

所以视觉统一的目标不是"做得更花"，而是"让用户更容易读"。

---

## 先统一页面宽度

视觉统一最先要处理的是宽度。

因为宽度会直接影响页面气质。

本站有三个主要区域：

| 区域 | 作用 |
|------|------|
| 左侧栏 | 学习档案、文章目录 |
| 中间栏 | 首页文章预览或文章正文 |
| 右侧栏 | Learning Stages、Tool Index、最近文章 |

如果这些宽度每个页面都不一样，用户从首页进入文章页时会明显感觉跳了一下。

所以宽度应该抽成统一变量。

例如：

```css
:root {
  --layout-sidebar-width: 320px;
  --layout-content-width: 820px;
  --layout-gap: 32px;
}
```

然后首页和文章页都使用这套变量。

这样改一个地方，全站布局都会同步。

这比在多个选择器里写死 `320px`、`820px` 更稳。

---

## 中间正文要克制

文章正文不是越宽越好。

太宽会让每一行文字过长，阅读会变累。

太窄又会让表格、代码块和图片没有空间。

所以中间栏需要一个平衡。

本站把正文宽度控制在一个适合长文阅读的范围。

这样有几个好处：

1. 段落行长比较舒服
2. 表格和代码块有足够空间
3. 左右侧栏还能保留导航功能
4. 首页文章卡片和文章正文宽度一致

宽度统一之后，首页和文章页会有明显的一体感。

用户从首页点击文章时，不会感觉页面结构突然变了。

---

## 侧栏要像工具，不要像装饰

侧栏不是为了把页面填满。

它承担很具体的功能：

- 学习档案提供整体状态
- 文章目录提供当前文章结构
- Learning Stages 提供学习主线
- Tool Index 提供问题查找
- 最近文章提供更新入口

所以侧栏的视觉应该安静。

不要做太重的颜色。

不要给每个模块加很夸张的装饰。

更好的方式是：

- 白色背景
- 细边框
- 轻阴影
- 标题字号适中
- 列表间距清楚
- 当前项和 hover 状态有轻微反馈

侧栏应该像一个可靠的工具面板。

用户需要它时能快速找到。

用户专心看正文时，它又不会抢走注意力。

---

## 卡片风格要统一

本站里有很多卡片：

- 学习档案卡片
- 文章预览卡片
- 阶段模块
- 工具模块
- 最近文章模块
- 文章正文卡片

如果每种卡片都单独设计，页面会很乱。

所以卡片应该共享一套基础语言：

| 属性 | 建议 |
|------|------|
| 背景 | 白色 |
| 边框 | 很浅的灰色 |
| 圆角 | 克制，不要过圆 |
| 阴影 | 轻微，用来和背景分层 |
| 内边距 | 保持一致 |
| 标题 | 字重清楚，但不要过大 |

注意，卡片不是越多越好。

卡片的作用是把信息分组。

如果一个卡片里再套很多卡片，页面会显得碎。

所以布局上要避免"卡片套卡片"。

真正需要分组的模块用卡片，其余地方保持干净。

---

## 链接颜色要稳定

内容站里，链接是非常重要的视觉语言。

用户需要一眼看出：

> 哪些文字可以点击？

所以链接颜色应该统一。

本站使用蓝色作为主要链接颜色。

它出现在：

- 首页文章标题
- 继续阅读
- Learning Stages 文章链接
- Tool Index 文章链接
- 最近文章标题
- 正文里的链接
- 文章目录链接

这样用户会形成稳定预期：

> 蓝色文字通常可以点。

不要一会儿蓝色，一会儿紫色，一会儿黑色加下划线。

链接样式越稳定，用户越不用思考。

---

## 正文样式要为阅读服务

文章页的正文样式最重要。

它要照顾各种 Markdown 元素。

比如：

| 元素 | 视觉要求 |
|------|----------|
| h2 / h3 | 层级清楚，和段落有足够间距 |
| 段落 | 行高舒适，文字颜色不要太黑 |
| 引用 | 和正文区分开，适合提示重点 |
| 列表 | 缩进清楚，行距稳定 |
| 表格 | 边框轻，内容可以换行 |
| 代码块 | 背景区分，不能撑破正文 |
| 图片 | 自适应正文宽度 |

正文样式不能只看短文。

一定要用长文、表格、代码块、图片一起测试。

因为真正会暴露问题的，往往是复杂内容。

比如：

- 长链接把正文撑宽
- 代码块横向溢出
- 表格在小屏幕上挤爆
- 图片超过卡片宽度

这些都要在 CSS 里提前处理。

---

## sticky 侧栏要克制使用

桌面端阅读长文时，侧栏 sticky 很有用。

文章往下滚动时，文章目录和右侧索引还能留在视野里。

这会让阅读更安心。

但 sticky 也要克制。

需要注意：

1. 顶部要留出导航栏高度
2. 侧栏不能比视口高太多而无法滚动
3. 移动端不要强行 sticky
4. 内容很多时侧栏自己要能滚动

所以可以给桌面端侧栏设置：

```css
position: sticky;
top: 84px;
max-height: calc(100vh - 112px);
overflow-y: auto;
```

这样侧栏既能固定，又不会在内容过多时被截断。

移动端则恢复普通文档流。

---

## 响应式不是压缩三栏

移动端不能把三栏硬挤到一起。

小屏幕上更适合单列。

顺序可以是：

1. 顶部导航
2. 主要内容
3. 学习档案
4. Learning Stages
5. Tool Index
6. 最近文章

或者根据实际页面做微调。

重点是：

> 移动端不是桌面端缩小版，而是重新安排阅读顺序。

字体不要随着视口无限缩放。

按钮和链接要有足够点击面积。

卡片之间要有稳定间距。

文字不能挤出容器。

这些比复杂动画更重要。

---

## 如何让 AI 帮你做视觉统一

你可以这样给 AI 任务：

```text
请完成 M6：视觉统一。

目标：
参考清爽技术博客风格，统一首页、文章页、左侧栏、右侧栏、正文和卡片的视觉语言。

要求：
1. 首页和文章页使用同一套三栏宽度变量
2. 左右侧栏宽度一致，中间正文宽度稳定
3. 学习档案、文章正文、阶段索引、工具索引、最近文章使用统一卡片风格
4. 链接颜色和 hover 状态统一
5. Markdown 正文里的标题、段落、引用、表格、代码块、图片都要有稳定样式
6. 代码块、表格、长链接不能撑破正文宽度
7. 桌面端侧栏可以 sticky，移动端改为单列布局
8. 不新增复杂动画，不做营销式 hero
9. 完成后运行 npm run build，并检查首页和文章页
```

这段任务里最重要的是不要只说"优化样式"。

"优化样式"太模糊。

要告诉 AI：

- 哪些模块要统一
- 宽度怎么统一
- 哪些内容不能溢出
- 桌面端和移动端分别怎么处理
- 哪些东西不要做

这样 AI 才会朝正确方向改。

## 验收视觉统一

视觉统一完成后，可以按下面这些问题检查：

1. 首页和文章页的三栏宽度是否一致？
2. 左侧栏和右侧栏是否视觉平衡？
3. 文章正文宽度是否适合长文阅读？
4. 首页文章卡片和文章正文卡片是否属于同一套风格？
5. Learning Stages 和 Tool Index 是否样式统一？
6. 链接颜色和 hover 状态是否一致？
7. 表格是否会撑破正文？
8. 代码块是否会撑破正文？
9. 图片是否不会超过正文宽度？
10. 桌面端侧栏 sticky 是否正常？
11. 移动端是否能自然变成单列？
12. `npm run build` 是否通过？

这里有一个实用建议：

不要只看首页。

一定要同时看：

- 首页
- 普通文章页
- 有表格的文章页
- 有代码块的文章页
- 有图片的文章页
- 较窄屏幕

视觉统一不是某一屏好看。

它是各种内容都不散架。

## 常见问题

### 1. 要不要加很多动画？

第一版不建议。

这个网站是学习内容站，不是展示型官网。

轻微 hover 反馈就够了。

动画太多会干扰阅读。

### 2. 卡片圆角要不要很大？

不建议。

技术内容站更适合克制的圆角。

圆角太大容易变得偏可爱或偏营销。

小圆角、轻边框、轻阴影会更稳。

### 3. 为什么不要做大 hero？

因为本站首页的核心是内容入口，不是宣传转化。

大 hero 会挤掉文章和学习路径。

用户来这里是为了读内容，而不是看一张漂亮封面。

## 小结

这一篇做的是视觉统一。

它不是给网站加一层装饰，而是让结构、内容和阅读体验变得一致。

你需要记住这条线：

1. 视觉统一要在结构稳定后做
2. 首页和文章页要共享宽度变量
3. 中间正文要适合长文阅读
4. 侧栏要像工具面板，不要抢注意力
5. 卡片、链接、标题和正文样式要统一
6. 表格、代码块、图片和长链接不能破坏布局
7. 桌面端可以三栏 sticky，移动端要单列阅读

做到这里，网站就从"功能能跑"进入了"作品可读"。

> **视觉统一的目的，不是让页面更热闹，而是让用户更放心地阅读。**
