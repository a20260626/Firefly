> 环境装好了，但 AI 到底怎么用？
> 这一篇教你"跟 AI 说人话，AI 给你出代码"的核心方法。

---

前两个阶段帮你做完了两件事：

- **认知篇**：放下了"我不会写代码"的包袱，建立了编程思维，看清了 AI 时代的核心竞争力
- **起步篇**：装好了所有工具——浏览器、VS Code、终端、Git、Node.js，跑通了第一个页面

但你可能还有一个疑问没解决：**AI 工具这么多，我到底该用哪个？怎么跟它说，它才能给我想要的东西？**

这就是阶段三要解决的核心问题。

---

## 选哪个 AI 工具

目前主流的 AI 编程工具有三类。作为新手，你不需要全用，**先选一个入手就行**。

### 推荐路线：从最简单的开始

| 工具 | 适合谁 | 怎么用 | 推荐度 |
|------|--------|--------|--------|
| **Claude**（对话式） | 纯新手 | 打开网页，直接说需求 | ⭐⭐⭐⭐⭐ |
| **ChatGPT**（对话式） | 纯新手 | 同上，打开网页直接聊 | ⭐⭐⭐⭐ |
| **Cursor**（AI IDE） | 想深入 | 在编辑器里用 AI 写代码 | ⭐⭐⭐（进阶） |

**我的建议：先用 Claude 或 ChatGPT。**

它们不需要安装，打开浏览器就能用，而且它们生成的代码可以直接复制到 VS Code 里运行。等你熟悉了"跟 AI 对话写代码"这个流程，再换 Cursor 也不迟。

> 💡 **别在工具选择上纠结。**
>
> 先选一个能用的，把时间花在"跟 AI 对话"本身。等你熟练了，自然就知道哪个更适合你。

在本教程中，我用 **Claude** 来演示（因为它在编程能力上表现最好）。但 90% 的内容在 ChatGPT、豆包、Kimi 上同样适用。

## 核心方法：写好一段"需求描述"

跟 AI 说"帮我写个网站"——它会给你一个模板，但大概率不是你想要的。

跟 AI 说清楚你要什么——它才会给你惊喜。

### 一个烂 prompt 和一个好 prompt

❌ **烂 prompt：**

> "帮我写一个个人网站。"

AI 的结果：一个到处都是 placeholder（占位内容）的通用模板。因为你没说你是谁、网站用来干嘛、想要什么风格。

✅ **好 prompt：**

> "帮我写一个个人作品集网页，我是 UI 设计师，需要展示 4 个设计项目。页面顶部有导航栏（关于我 / 作品 / 联系），中间是项目展示区（每个项目一张图+标题+简短描述），底部是联系表单。风格简洁，白色背景，圆角卡片，字体用无衬线体。"

AI 的结果：一个结构清晰、风格明确、可以直接用的页面。因为你给了它足够的信息。

### Prompt 公式

把好的 prompt 拆开，其实有一个通用公式：

```text
[角色/背景] + [要做什么] + [包含什么内容] + [长什么样] + [特殊要求]
```

套用到上面的例子：

| 要素 | 内容 |
|------|------|
| 角色/背景 | "我是 UI 设计师" |
| 要做什么 | "个人作品集网页" |
| 包含什么 | "导航栏、4个项目展示、联系表单" |
| 长什么样 | "简洁、白色背景、圆角卡片、无衬线体" |
| 特殊要求 | — |

你不需要每次都写满 5 个要素，但写得越多，AI 给你的东西越接近你想要的。

## 第一轮对话：生成你的第一个页面

打开 Claude（或 ChatGPT），输入下面的 prompt。**不用理解每一句在说什么，先感受这个过程。**

```text
请帮我生成一个完整的 HTML 文件。

页面需求：
- 标题显示"我的第一个 AI 编程作品"
- 页面背景浅灰色，文字深灰色
- 中间有一个居中的白色卡片
- 卡片里显示：一个标题、一段介绍文字、一张示例图片（用占位图）
- 卡片下方有一个按钮，点击后弹出"你好，AI 世界！"的提示

请输出完整的 HTML 代码，并告诉我怎么在浏览器中查看。
```

AI 会返回一段代码。**现在你做这件事：**

1. 在 VS Code 中新建文件，命名为 `my-first-page.html`
2. 把 AI 给的完整代码**全部复制**进去
3. 按 **`` Ctrl + ` ``** 打开 VS Code 终端
4. 在文件上右键 → **"Open with Live Server"**

浏览器打开，你应该能看到一个页面——它就是 AI 根据你的描述生成的。

🎉 **这就是你用 AI 写的第一段代码。** 你没有写任何一行，但它是你的。

## 第二轮对话：不满意？告诉 AI 改

很少有人第一次就拿到完美结果。**但修改才是 AI 编程真正的优势所在。**

你不需要自己改代码——你只需要告诉 AI "哪里不对，改成什么样"。

### 修改对话示例

```text
页面整体不错，但有几点我想改：
1. 把浅灰色背景改成白色，卡片改成浅灰色
2. 按钮颜色改成蓝色
3. 在卡片里加一个"关于我"的小图标
4. 卡片圆角再大一点

帮我更新代码。
```

AI 会重新输出完整的 HTML。把更新后的代码覆盖到你的文件里，刷新浏览器就能看到变化。

> ⚠️ **重要：** 每次修改时，把 AI 输出的**完整新代码**全部复制替换，不要只替换片段。这样最不容易出错。

### 可以改什么

| 想改什么 | 对 AI 说 |
|---------|---------|
| 颜色 | "把主色调改成蓝色" |
| 布局 | "把两栏改成三栏" |
| 内容 | "把标题改成'我的作品集'" |
| 功能 | "加一个回到顶部按钮" |
| 风格 | "改成暗色模式" |

## 常见的坑（以及怎么绕过）

### 坑 1：一次说太多

❌ "帮我做一个完整的电商网站，包括商品展示、购物车、支付、后台管理、数据分析……"

✅ 先做商品展示页 → 再做购物车 → 再做支付。**一次只做一件事。**

> AI 编程的黄金法则：**把大需求切成小步骤，一次聊一个。**

### 坑 2：不说背景

❌ "帮我改好看一点。"——AI 不知道什么是你眼中的"好看"。

✅ "改成苹果官网那种风格，大量留白，细字体，极简。"

### 坑 3：代码出错了就慌了

❌ （报错了，不知道怎么办）

✅ 把错误信息**完整复制**给 AI，同时告诉它你做了什么操作。AI 能定位问题。

> 🛠️ **黄金救场三步：**
>
> 1. 复制全部错误信息
> 2. 告诉 AI 你刚才做了什么
> 3. 把相关代码发给它
>
> 然后问："哪里出错了？怎么修？"

### 坑 4：把 AI 当搜索引擎

❌ "Vue 和 React 哪个好？"——AI 会给泛泛的对比，但没什么用。

✅ "帮我用 Vue 写一个待办清单"——让 AI 用代码回答你。

> **AI 编程工具擅长"做出来"，不擅长"比较和推荐"。** 需要做决策时，先自己查资料，再用 AI 执行。

## 你的第一个"完整项目"

现在，用你刚刚学会的方法，做一个小项目来巩固：

> **用 AI 做一个"个人名片"网页，包含：**
>
> - 你的头像（用占位图）
> - 你的名字和一句话介绍
> - 三个技能标签（如：设计 / 写作 / AI 编程）
> - 一个"联系我"的链接
>
> 风格你决定，颜色你决定。不满意就改，直到你看顺眼为止。

**怎么做：**

1. 打开 Claude/ChatGPT
2. 输入你的需求（用上面学的 prompt 公式）
3. 把生成的代码存成 HTML 文件
4. 用 Live Server 预览
5. 不满意就继续对话修改
6. **完成了！**

这个小项目做完，你就走完了"提需求 → AI 生成 → 验收 → 修改 → 完成"的完整闭环。这个流程，后面所有项目都通用。

## 小结

| 你学到了 | 一句话总结 |
|---------|-----------|
| 选哪个工具 | 先用 Claude 或 ChatGPT，打开网页就能用 |
| 怎么写 prompt | [角色] + [做什么] + [包含什么] + [长什么样] |
| 第一轮对话 | 描述需求 → AI 出代码 → Live Server 预览 |
| 第二轮对话 | 告诉 AI "哪里不对，改成什么样" → 替换代码 → 刷新 |
| 遇到报错 | 复制错误信息给 AI，问它怎么修 |
| 批量修改 | 一次只改一个点，逐步迭代 |

你已经学会了跟 AI 协作写代码的核心方法。下一阶段，我们会用它来做一个真正的、完整的网页——也就是这个网站本身。
