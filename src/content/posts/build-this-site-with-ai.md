> 这是"用 AI 做出这个网页"的第一篇实战记录。
> 真正动手写代码之前，我们先做一件更重要的事：写一个简单 PRD。

---

前面三个阶段，我们已经完成了准备工作：

- **认知篇**：知道 AI 编程不是魔法，而是一种新的协作方式
- **起步篇**：装好了编辑器、终端、Git、Node.js 等基础工具
- **对话篇**：学会了怎么把需求说给 AI 听

现在进入阶段四：实战篇。

这一次，我们要做的项目不是凭空编出来的练习题，而是你正在看的这个网站本身。

但在让 AI 写代码之前，先别急。

真正的第一步不是：

> "帮我写一个网站。"

而是：

> **先写一个简单 PRD，把你到底想做什么讲清楚。**

---

## PRD 是什么

PRD 是 Product Requirements Document 的缩写，中文通常叫"产品需求文档"。

听起来很正式，但你现在不需要写公司里那种几十页的复杂文档。

对新手来说，PRD 只需要回答一个问题：

> **我要做什么，为什么做，做成什么样才算完成？**

这句话很重要。

因为 AI 写代码很快，但它不会自动知道你的目标、审美、内容结构和取舍标准。

如果你只说：

> "帮我做一个学习网站。"

AI 会给你一个很泛的模板。

但如果你说：

> "我要做一个 AI 编程学习笔记站，给零基础用户看。它要有 6 个学习阶段、5 类工具索引、首页文章预览、文章阅读页和 Markdown 内容系统。视觉参考 Sukka's Blog，浅灰背景、白色卡片、三栏布局、蓝色链接，整体要像一个长期维护的博客。"

AI 的输出就会准很多。

这就是 PRD 的作用：**把模糊想法变成可执行的说明书。**

---

## 为什么不要一上来就写代码

很多人第一次用 AI 编程，会直接把一句大需求丢给 AI：

```text
帮我做一个好看的个人网站。
```

AI 当然能写。

但问题是，它不知道：

- 这个网站给谁看
- 首页应该展示什么
- 有几种内容类型
- 要不要文章页
- 要不要分类
- 要不要适配手机
- 什么叫"好看"
- 参考风格是什么

于是它只能猜。

你会得到一个"看起来还可以"的页面，但很快会遇到问题：

1. 想加文章，不知道放哪
2. 想改布局，结构很乱
3. 想做分类，数据没设计
4. 想继续扩展，发现之前生成的代码不好维护

所以，PRD 的意义不是让你变得很专业。

它的意义是让 AI 少猜一点，让你自己也少返工一点。

> **写 PRD 不是为了写文档而写文档，而是为了让后面的代码有方向。**

---

## 一个简单 PRD 需要哪些内容

新手写 PRD，不需要追求完整模板。

你只要写清楚 6 件事就够了。

| 模块 | 要回答的问题 |
|------|--------------|
| 1. 项目目标 | 这个网站要解决什么问题？ |
| 2. 目标用户 | 谁会来看这个网站？ |
| 3. 核心页面 | 至少需要哪些页面或区域？ |
| 4. 内容结构 | 文章、分类、工具索引怎么组织？ |
| 5. 视觉参考 | 希望它看起来像什么？ |
| 6. 完成标准 | 做到什么程度算第一版完成？ |

你可以把它理解成一个"项目说明卡"。

它不用长，但要清楚。

## 用这个网站举例

我们现在这个网站的 PRD，可以这样写。

### 1. 项目目标

做一个面向零基础用户的 AI 编程学习笔记站。

它不是单纯展示文章，而是要把学习路径组织起来，让用户知道：

- 先理解什么
- 再安装什么
- 然后怎么和 AI 对话
- 接着怎么实战
- 出错后怎么调试
- 最后怎么发布

一句话版本：

> 帮零基础用户从"不敢碰代码"走到"能用 AI 做出并发布一个网页"。

### 2. 目标用户

这个网站不是给资深工程师看的。

它面向的是：

- 对 AI 编程感兴趣，但没有系统学过代码的人
- 想做个人网页、作品集、学习笔记站的人
- 会使用电脑，但对终端、Git、部署比较陌生的人
- 希望 AI 能帮自己把想法变成网页的人

这会直接影响写作语气和页面设计。

所以文章里要尽量少用黑话，多用类比，多告诉用户"为什么要这么做"。

### 3. 核心页面

第一版不需要做很多页面。

只需要两个核心状态：

| 页面状态 | 内容 |
|----------|------|
| 首页 | 学习档案、文章预览、6 个阶段、5 类工具索引、最近文章 |
| 文章页 | 学习档案、文章目录、正文、Learning Stages、Tool Index、最近文章 |

这里有一个重要取舍：**先不做复杂后台。**

文章直接用 Markdown 文件管理，足够简单，也适合学习项目。

### 4. 内容结构

这个网站的内容按两条线组织。

第一条线是 **6 个学习阶段**：

1. 认知篇：ai编程能做什么
2. 起步篇：环境从0到1
3. 对话篇：用ai写第一行代码
4. 实战篇：从网页开始
5. 调试篇：翻车与修复
6. 发布篇：让世界看到

第二条线是 **5 类工具索引**：

1. ai编程助手：Codex / Claude / GPT
2. ai IDE/编辑器：Cursor / VS Code
3. 版本控制：Git / GitHub
4. 前端基础：HTML / CSS / JavaScript
5. 部署发布：GitHub Pages / Vercel / Netlify

为什么要有两条线？

因为用户有两种阅读方式：

- 按学习阶段一步步读
- 按工具类别快速查

这就是信息架构。

### 5. 视觉参考

这个网站参考的是 Sukka's Blog 的博客风格。

我们想要的不是花哨，而是：

- 浅灰背景
- 白色卡片
- 细边框
- 轻阴影
- 蓝色链接
- 三栏布局
- 正文宽度克制
- 侧栏提供辅助索引

一句话描述：

> 像一个安静、清晰、适合长期阅读和维护的技术博客。

这句话比"做得好看一点"有用得多。

### 6. 完成标准

第一版完成标准不要太大。

可以这样定：

1. 首页能看到文章预览
2. 点击标题或继续阅读能进入文章页
3. 文章页能显示 Markdown 正文
4. 左侧有学习档案和文章目录
5. 右侧有 Learning Stages、Tool Index 和最近文章
6. 每篇文章能按阶段和工具自动归类
7. 页面在桌面和手机上都能正常阅读

做到这些，第一版就算完成。

后面再慢慢加搜索、标签、评论、部署、SEO。

## 一个可直接复制的简单 PRD 模板

你以后做自己的项目，可以直接复制下面这个模板。

```text
# 项目名称

用一句话描述这个项目是什么。

## 1. 项目目标

这个项目要解决什么问题？
用户打开它以后，能完成什么事？

## 2. 目标用户

这个项目给谁用？
他们现在遇到什么困难？
他们的技术水平大概怎样？

## 3. 核心页面 / 模块

列出第一版必须有的页面或模块。
先写必须做的，不要把所有想法都塞进去。

## 4. 内容结构

内容怎么分类？
有哪些列表、卡片、文章、标签或索引？

## 5. 视觉风格

参考哪个网站或产品？
希望整体感觉是什么？
颜色、布局、卡片、字体有没有明确要求？

## 6. 交互要求

用户点击哪里？
跳转到哪里？
哪些内容需要展开、筛选或切换？

## 7. 第一版完成标准

做到哪几件事，就算第一版完成？
哪些功能可以以后再做？
```

这个模板不复杂，但已经足够让 AI 开始认真工作。

## 把 PRD 发给 AI 时，应该怎么说

写完 PRD 以后，不要直接说"照着做"。

更好的方式是让 AI 先帮你检查和拆解。

你可以这样说：

```text
下面是我的网站 PRD。
请先不要写代码。

请帮我做三件事：
1. 检查这个 PRD 有没有不清楚或冲突的地方
2. 把它拆成可执行的开发步骤
3. 告诉我第一步应该先做什么

[把 PRD 粘贴在这里]
```

这一步很关键。

因为 AI 可能会发现你没写清楚的地方，比如：

- 首页和文章页是不是同一个布局？
- 文章数据从哪里来？
- 移动端怎么展示三栏？
- 点击文章后是弹窗还是新页面？

如果这些问题在写代码前解决，后面会顺很多。

---

## 这个网站的第一步是什么

根据这个项目的 PRD，第一步不是写文章，也不是调颜色。

第一步应该是：

> **搭项目基础和数据结构。**

也就是先把这些事情定下来：

1. 用什么前端框架
2. 文章数据怎么存
3. 阶段和工具索引怎么定义
4. 首页和文章页怎么切换
5. Markdown 怎么渲染

这就是为什么我们前面先做了 M1：项目基础。

因为没有基础结构，后面的页面越做越乱。

有了基础结构，后面加文章、改布局、调样式，就都变成了可控的小步骤。

---

## PRD 不需要一次写完美

还有一个很重要的心态：

> **PRD 不是圣旨，它是第一版地图。**

你一开始写的 PRD 一定会变。

这个网站就是这样。

一开始我们只想做学习路线和工具索引。

后来又逐步加入了：

- 首页文章预览
- 点击标题跳转文章
- 文章目录
- 最近文章
- Learning Stages 文章链接
- Tool Index 文章链接
- 三栏宽度参考 Sukka's Blog
- 文章页和首页保持同一布局节奏

这些变化不是坏事。

它们说明项目在变清楚。

真正重要的是：每次变化都要说清楚，然后让 AI 帮你有边界地修改。

---

## 小结

阶段四的第一步，是学会写一个简单 PRD。

你不需要写得很专业，但至少要讲清楚：

1. 项目目标
2. 目标用户
3. 核心页面
4. 内容结构
5. 视觉参考
6. 交互要求
7. 第一版完成标准

有了 PRD，AI 才知道它在帮你做什么。

没有 PRD，AI 只能猜。

而 AI 编程最重要的能力，恰恰不是"让 AI 猜得更准"，而是：

> **把你想要的东西，说到足够清楚。**

下一篇，我们就可以拿着这个 PRD，开始拆任务、搭项目基础，并一步步做出这个网页。
