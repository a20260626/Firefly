前面三篇文章聊了很多"为什么"和"是什么"。

从这篇开始，我们不聊了——**开始动手。**

这一阶段的目标很简单：在你的电脑上装好所有需要的东西，然后跑通第一行代码。整个过程大概需要 30 分钟，你不需要懂任何技术细节，跟着步骤走就行。

我们把它拆成 5 个关卡，像玩游戏一样，一个一个解锁。

---

## 关卡总览

| 关卡 | 解锁什么 | 为什么要装 |
|------|---------|-----------|
| 关卡 1 | **浏览器** | 你本来就有，但有些设置能让你效率翻倍 |
| 关卡 2 | **VS Code（编辑器）** | 写代码的地方，你的主战场 |
| 关卡 3 | **终端（命令行）** | 跟电脑"说命令"的地方，比鼠标快 10 倍 |
| 关卡 4 | **Git + GitHub** | 给你的代码装一个"时光机" |
| 关卡 5 | **Node.js** | 让你的网页能在本地跑起来 |

每一个关卡我都会告诉你：**装什么、在哪装、装完怎么验证它好了。** 不会让你装完也不知道装了啥。

## 关卡 1：浏览器

做网页开发，推荐使用 **Chrome**。如果你电脑上已经有了，可以直接跳到后面的"开发者工具"部分。

如果你只有 Edge（Windows 系统自带），没关系，我们用 Edge 来下载 Chrome——**正好体验一下"用浏览器下载工具"这个操作流程。**

### 用 Edge 下载 Chrome

1. 打开桌面上 Edge 浏览器（那个蓝色/绿色的"e"图标）
2. 在顶部地址栏输入以下网址，按回车：

   ```text
   https://www.google.com/chrome/
   ```

3. 页面中间有一个大的蓝色按钮，写着 **"下载 Chrome"**，点击它
4. 弹出下载窗口，点击 **"保留"** 或 **"保存"**（不同版本的 Edge 按钮文字可能略有不同）
5. 等待下载完成（一般几秒钟）
6. 下载完成后，打开下载的文件（通常在左下角或下载文件夹里）
7. 会弹出一个安装向导，点击 **"是"**（允许安装），然后等待安装进度条走完
8. 安装完成后，Chrome 会自动打开

> 💡 如果下载页面是英文的，找 **"Download Chrome"** 按钮，点进去之后按照步骤走就好。

### 把 Chrome 设为常用浏览器

1. 打开 Chrome
2. 右上角三个点（菜单）→ **"设置"**
3. 左侧选 **"默认浏览器"**
4. 点击 **"设为默认浏览器"** 按钮
5. 系统会弹出一个设置窗口，在"Web 浏览器"一栏中选择 **Google Chrome**

以后你点链接就会默认用 Chrome 打开了。

### 打开开发者工具

现在，我们做一个重要的小设置——**打开开发者工具。**

这个工具是你以后跟 AI 协作调试网页时的"眼睛"：

1. 在 Chrome 里任意一个网页上右键 → 选择 **"检查"**（或者直接按键盘上的 **F12**）
2. 会弹出一个面板，有 Elements（元素）、Console（控制台）、Network（网络）等标签
3. 你先不用理解它们具体是什么，只需要记住一件事：

> **以后 AI 让你"打开开发者工具看报错"时，按 F12 就行。**

🔓 **关卡 1 完成。**

## 关卡 2：VS Code（编辑器）

这是你写代码的主阵地。相当于 Word 之于写文章，Excel 之于做表格。

### 下载安装

1. 打开浏览器，搜索 **"VS Code 下载"** 或直接访问 [code.visualstudio.com](https://code.visualstudio.com)
2. 点击蓝色的 **Download for Windows** 按钮
3. 下载完成后双击安装，一路点"下一步"就行
4. 安装成功后打开，你会看到这样的界面：

![VS Code 界面](https://code.visualstudio.com/assets/docs/getstarted/userinterface/hero.png)

### 汉化（可选）

VS Code 默认是英文的。如果你看英文界面不习惯：

1. 打开 VS Code
2. 点击左侧边栏的四个方块图标（Extensions，扩展商店）
3. 在搜索框输入 `Chinese`
4. 找到 **"Chinese (Simplified) Language Pack"**，点 Install
5. 装完后右下角会弹出提示，点 **Restart** 重启 VS Code
6. 界面就变成中文了

### 安装一个超级重要的插件

回到扩展商店（左边四个方块那个图标），搜索并安装以下插件——它们是 AI 编程的"队友"：

**必装：**

- **GitLens** — 让你的 Git 操作可视化，后面会用到
- **Prettier** — 自动帮你把代码排版变整齐
- **Live Server** — 一键在浏览器里预览你的网页

> 💡 **怎么装：** 搜索到插件 → 点 Install → 等几秒装好。就这么简单。

### 验证

打开 VS Code，在左侧文件区域右键 → 新建文件 → 命名为 `hello.html` → 在文件中输入：

```html
<h1>你好，世界！</h1>
```

然后右键这个文件 → 选择 **"Open with Live Server"**。浏览器会自动打开，你应该能看到一行大字：**你好，世界！**

如果能，说明 VS Code 安装正确，Live Server 也正常工作。

🔓 **关卡 2 完成。**

## 关卡 3：终端（命令行）

"终端"这个名字听起来很吓人。但你回想上一篇文章说的——**编程思维就是拆解问题**。终端不过是一个让你用"打字"而不是"点鼠标"来给电脑下指令的地方。

举个例子：

- 鼠标操作：右键 → 新建文件夹 → 输入名称 → 回车
- 终端操作：输入 `mkdir my-project` → 回车

后者更快，而且可以告诉 AI"帮我在终端运行这个命令"，AI 会直接给你命令文本，你复制粘贴就行。

### Windows 上的终端

如果你用 Windows 10 或 11，系统自带了**Windows Terminal**（推荐）或 **PowerShell**。

按 `Win + R`，输入 `cmd` 回车——这就是最基础的终端。也可以用 VS Code 内置终端：在 VS Code 中按 **`` Ctrl + ` ``**（反引号），底部会打开一个终端面板。

> 💡 **以后你 99% 的时间会在 VS Code 里用终端**，不需要专门打开另一个窗口。

### 几个你马上会用到的命令

| 命令 | 作用 | 类比 |
|------|------|------|
| `dir` | 列出当前文件夹的内容 | 相当于打开文件夹看里面有什么 |
| `cd 文件夹名` | 进入某个文件夹 | 双击文件夹 |
| `cd ..` | 返回上一级 | 点"返回"按钮 |
| `mkdir 名字` | 新建文件夹 | 右键→新建文件夹 |
| `code .` | 用 VS Code 打开当前文件夹 | — |

不需要背。用到的时候查一下，或者在用 AI 的时候直接问它"这个命令是什么意思"。

🔓 **关卡 3 完成。**

## 关卡 4：Git + GitHub

Git 是一个"版本控制工具"——你可以理解为它是代码的"时光机"。

GitHub 是一个"代码存盘的地方"——你在 Git 上存的代码可以同步到 GitHub，这样换电脑也能继续，而且别人也能看到你的作品。

### 安装 Git

1. 搜索 **"Git for Windows 下载"** 或直接访问 [git-scm.com/downloads/win](https://git-scm.com/downloads/win)
2. 下载后双击安装，**一路默认选项即可**
3. 安装完成后，在终端（或 VS Code 终端）中输入：

```bash
git --version
```

如果能看到类似 `git version 2.x.x` 的输出，说明安装成功。

### 配置你的名字和邮箱

打开终端，输入以下两行（把名字和邮箱换成你的）：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

这只是为了让 Git 知道"谁在操作"，跟 GitHub 账号没有直接关系。

### 注册 GitHub

1. 打开 [github.com](https://github.com)
2. 点击 **Sign up**，输入用户名、邮箱、密码
3. 验证邮箱
4. 完成！你现在有了一个"代码存盘"的云空间

### 验证 Git 工作

打开 VS Code 终端，输入：

```bash
git version
```

能显示版本号就对了。

🔓 **关卡 4 完成。**

## 关卡 5：Node.js

Node.js 是一个让 JavaScript（前端语言）能在电脑上跑起来的"引擎"。很多 AI 生成的网页项目都需要它来运行。

### 下载安装

1. 访问 [nodejs.org](https://nodejs.org)
2. 你会看到两个版本，选左边那个 **LTS（长期支持版）** 下载
3. 双击安装，一路默认选项
4. 安装完成后，打开终端（或 VS Code 终端），输入：

```bash
node --version
```

能看到类似 `v18.x.x` 的输出，说明安装成功。

再输入：

```bash
npm --version
```

能看到 npm（Node.js 自带的包管理器）的版本号。

> 💡 **npm 是什么？** 你可以把它理解成"手机应用商店"——全世界的程序员把写好的功能模块上传到 npm，你需要的时候用一行命令就能下载使用。后面实战阶段你会经常用到它。

🔓 **关卡 5 完成。**

## 终极验证：跑通你的第一个项目

5 个关卡都解锁后，我们来验证一下整个环境是不是能正常协作。

1. 打开 VS Code
2. 按 **`` Ctrl + ` ``** 打开终端
3. 依次输入以下命令（每输入一行按一次回车）：

```bash
mkdir my-first-project
cd my-first-project
echo "<h1>我的第一个 AI 编程项目</h1>" > index.html
code .
```

4. 在 VS Code 中右键 `index.html` → 选择 **"Open with Live Server"**

如果浏览器打开并显示 **"我的第一个 AI 编程项目"**——恭喜，你的全套工具链已经跑通了！

## 你的工具箱总览

| 工具 | 作用 | 什么时候用到 |
|------|------|------------|
| 浏览器 + F12 | 看网页效果、查报错 | 每次调试都用 |
| VS Code | 写代码、管理文件 | 每天打开 |
| 终端（VS Code 内置） | 运行命令 | 装东西、跑项目 |
| Git | 保存版本历史 | 每完成一个功能保存一次 |
| GitHub | 云上存代码、展示作品 | 发布、分享 |
| Node.js + npm | 运行和安装项目依赖 | 每次跑项目时 |
| Live Server | 实时预览网页 | 每次做网页时 |

你不需要记它们每一个的用法。**你只需要知道"我有这个工具"**——当 AI 跟你说"在终端运行 npm install"时，你知道终端在哪；当 AI 说"用 Live Server 预览"时，你知道右键就有。
